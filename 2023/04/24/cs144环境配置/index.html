cs144环境配置

发表于2023-04-24|更新于2023-04-25|cs144
|阅读量:

又是究极折磨的一天

使用系统：自己云服务器上部署的docker容器（ubuntu 22.04）

这里部署docker容器也是花了半天时间（菜哭了）

最开始用的ubuntu 20.04 但是到lab第二个cmake的时候出现问题（第一次）

这时候不知道是什么问题，但是好像在lab0文档里面看到了ubuntu 22的字眼，所以准备把这个容器废掉换新的…

但是我没注意到是22.10，docker pull下来ubuntu 22.04镜像重新配置

另外在这个翻来覆去换容器的过程中（没错我换了好几次…），我发现有很多初始配置是相同的，所以我将这个最原始的ubuntu 22.04镜像稍作配置commit成了我的u22:1.1镜像（早该这么干了！），写在【另外一篇博客里了，待更新】

然后跟着lab0的文档重开

这次到了第一个cmake的时候就出问题了（那我第一次是怎么成功的？！）

但是还好，提示我用的cmake版本太低了，那就升级呗

然后我开始google…（菜）

第一次尝试用pip安装，其实更新成功了，而且很简单

然而我真的要被自己笨哭了，第一个cmake -S . -B build过了，但是第二个cmake --build build没过，和开头报一样的错。我觉得会不会是pip安装cmake有问题…然后又重开了（到这里应该已经重开容器四五次了，开始发疯），后来才知道这里其实不是cmake的问题

第二次参考这个教程和这个教程

先从cmake官网下载.tar.gz，然后解压

sudo mv 到/usr/local/下面

创建链接 sudo ln -s

最后查看cmake --version，更新成功

cmake -S . -B build 成功了..

但是cmake --build build还是报一样的错，这个时候我真的要疯掉了

我已经不知道怎么形容当时的心情了

万念俱灰，我去群里问了一下，有个老哥跟我说是gcc版本的问题，我去查了一眼cs144配环境的指导发现果然要至少gcc12

我查了一下我用的gcc11，果然有问题！

开始尝试换成gcc12

这里有另外一个老哥让我问gpt，我去问了一下，最后确实有提到编译器是否支持c++版本的问题

然后换成gcc12的过程比较简单，问gpt就完事了，本地可以有多个gcc版本，把gcc12优先级调到最高即可

但是

换成gcc12还是不行，依旧报错

20分钟之后我无奈地翻着terminal，发现

这里用 cmake -S . -B build 之后第一行信息显示11.3.0 我查了一下之前安装的gcc果然是11.3.0

说明根本没换成12？！为什么？

不对啊，默认gcc确实换成gcc12了

我这里再次去问gpt，然后得到有关**g++**的字眼

我尝试去看g++ 版本，果然还是11.3！

然后在gpt帮助下换成12.1（其实跟gcc一样不过这里有点小插曲）

接着

重新cmake -S . -B build

cmake --build build

太不容易了！！真的尝到了历经千难万阻之后解决问题的成就感和满足感

多问ChatGPT！
